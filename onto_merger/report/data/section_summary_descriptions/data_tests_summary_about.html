Great Expectations is a tool for validating and documenting data to maintain quality. The OntoMerger pipeline tests all input and output data. Data tests check node ID schema, table schema and content, according to values specified in the configuration JSON (e.g. are all the mapping types specified in the JSON, are all node IDs adhere to the schema etc.). Data tests (expectations) and results (validations) can be examined via the data docs HTML report. For more details about how we use data tests, please read the docs.
