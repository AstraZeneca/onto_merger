The output consists of four tables:

  • nodes.csv: contains the final set of nodes (i.e. canonical nodes), excluding merged nodes.
  • merges.csv: contains the aligned and the corresponding canonical node IDs; this set is the difference between the input and output nodes.
  • mappings.csv: node IDs are potentially updated with canonical node IDs and/or internal code reassignments
  • edges_hierarchy.csv: contains the merged ontology hierarchy.

All output tables are profiled and validated, these details are presented in the Data Profiling and Data Tests sections, respectively. For more details about the output data, please read the docs.
